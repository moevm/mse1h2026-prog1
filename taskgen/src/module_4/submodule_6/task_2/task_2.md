# Обработка ошибок: perror


### Задание №2
- **Сложность:** легко  
- **Формулировка:**  
  Ниже приведены 4 фрагмента кода (1–4) и 4 строки вывода (A–D).  
  Сопоставьте каждый фрагмент с тем выводом, который он произведёт в `stderr`.  
  Введите ответ в виде 4 букв без пробелов — i-я буква соответствует фрагменту i.  
  Например: `BCAD` означает: фрагмент 1 → B, фрагмент 2 → C, фрагмент 3 → A, фрагмент 4 → D.

  **Фрагменты кода:**

  **Фрагмент 1:**
  ```c
  errno = ENOENT;
  perror("error");
  ```

  **Фрагмент 2:**
  ```c
  errno = EACCES;
  perror("error");
  ```

  **Фрагмент 3:**
  ```c
  errno = EINVAL;
  perror("error");
  ```

  **Фрагмент 4:**
  ```c
  errno = ERANGE;
  perror("error");
  ```

  **Варианты вывода (A–D) перемешаны согласно `seed`:**  
  Порядок вариантов A–D определяется значением `seed % 24` (перестановка 4 элементов).

  | Метка | Строка вывода |
  |-------|---------------|
  | ? | `error: No such file or directory` |
  | ? | `error: Permission denied` |
  | ? | `error: Invalid argument` |
  | ? | `error: Numerical result out of range` |

  Метки A, B, C, D назначаются строкам в порядке, определяемом `seed % 24`.

- **Параметры задания:**  
  Правильное сопоставление (до перемешивания меток):

  | Фрагмент | `errno` | Правильный вывод |
  |----------|---------|-----------------|
  | 1 | `ENOENT` | `error: No such file or directory` |
  | 2 | `EACCES` | `error: Permission denied` |
  | 3 | `EINVAL` | `error: Invalid argument` |
  | 4 | `ERANGE` | `error: Numerical result out of range` |
