# Подмодуль 6: Компиляторы и флаги

Подмодуль посвящён управлению поведением компилятора `gcc` с помощью флагов. Рассматриваются стандарты языка (`-std=cXX`, `-std=gnuXX`) и GNU-расширения, уровни оптимизации кода (`-O0`...`-O3`, `-Os`), а также инструменты отладки и статического/динамического анализа. Особое внимание уделяется включению предупреждений (`-Wall`, `-Wextra`), преобразованию их в ошибки (`-Werror`) для обеспечения качества кода, генерации отладочной информации (`-g`) и применению санитайзеров (`-fsanitize`) для поиска ошибок работы с памятью и неопределённого поведения во время выполнения.

## Темы заданий

| № | Название | Тема |
|---|----------|------|
| 6.1.1 | Флаги `-Wall` \| `-Wextra` | Написать команду компиляции с включением базовых и расширенных предупреждений |
| 6.2.1 | Флаг `-std` | Выбрать флаг для строгого соответствия заданному стандарту C |
| 6.3.1 | GNU-расширения | Выбрать флаг для использования стандарта C с поддержкой расширений GCC |
| 6.4.1 | Флаги оптимизации `-O<число>` | Составить команду компиляции по текстовому описанию уровня оптимизации |
| 6.5.1 | Флаг `-g` | Написать команду для добавления отладочной информации в исполняемый файл |
| 6.6.1 | Санитайзеры | Написать команду с включением санитайзера и отладочной информации |
| 6.7.1 | Предупреждения и ошибки | Написать команду сборки с флагом `-Werror` (превращение предупреждений в ошибки) |
