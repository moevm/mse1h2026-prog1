# Подмодуль 3: Заголовочные файлы

Подмодуль посвящён организации кода через заголовочные файлы в языке Си. Рассматриваются проблемы повторного включения и методы защиты: классические макросы (`#ifndef`/`#define`) и нестандартная директива `#pragma once`.
Изучаются принципы модульной сборки (1 `.h` + N `.c` файлов, где N ≥ 1), правила размещения кода в заголовках (что можно и нельзя включать), а также корректное использование `extern` для глобальных переменных и `inline` для функций в `.h`.

## Темы заданий

| № | Название | Тема    |
|---|----------|---------|
| 3.1.1 | Классический include guard | Написать защиту от повторного включения для заданного `.h` файла |
| 3.2.1 | Директива `#pragma once` | Выбрать корректные свойства и особенности директивы `#pragma once` |
