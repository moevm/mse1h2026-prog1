# Подмодуль 4: Линковка

Подмодуль посвящён этапу линковки программы на языке Си. Рассматриваются понятия символов, которые обрабатывает линковщик, и различие между внешней и внутренней линковкой. Изучается влияние ключевого слова `static` на область видимости на уровне файла, а также разбор и устранение типичных ошибок компоновщика: множественного определения (`multiple definition`) и неразрешённой ссылки (`undefined reference`).

## Темы заданий

| № | Название | Тема |
|---|----------|------|
| 4.1.1 | Символы | Определить имена, обрабатываемые линковщиком, в заданном фрагменте кода |
| 4.2.1 | Внешняя линковка | Написать корректное объявление глобальной переменной через `extern` |
| 4.3.1 | Внутренняя линковка | Определить глобальную переменную с внутренней линковкой (`static`) |
| 4.4.1 | `static` на уровне файла | Написать определение вспомогательной функции с внутренней линковкой |
| 4.5.1 | Ошибка `multiple definition` | Исправить дублирующееся определение переменной через объявление |
| 4.6.1 | Ошибка `undefined reference` | Составить корректную команду `gcc` для подключения недостающего файла реализации |
